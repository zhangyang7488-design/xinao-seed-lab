## 3. 省 token 栈 — RTK · Caveman · OmniRoute（对你很重要）

关系不是「谁启发谁」，而是 **OmniRoute 把 RTK + Caveman 堆进网关**：

┌─────────────┬──────────────────────────────────────────────────────────────┐
│ 组件        │ 干什么                                                       │
├─────────────┼──────────────────────────────────────────────────────────────┤
│ RTK         │ Rust CLI 代理；压缩 git diff / pytest / grep 等工具输出      │
│ (rtk-ai/rtk)│ 进模型 context 之前；宣称 60–90% 工具侧 token 省              │
│             │ `rtk init -g --codex` 可 hook Codex                          │
├─────────────┼──────────────────────────────────────────────────────────────┤
│ Caveman     │ JuliusBrussee 技能；多条填充/精简规则（lite/full/ultra/wenyan）│
│             │ 压模型**输出**废话，代码/命令/错误原样保留；~65% output 省    │
│             │ caveman-compress 还能压 CLAUDE.md 等记忆文件 ~46% input 省   │
├─────────────┼──────────────────────────────────────────────────────────────┤
│ OmniRoute   │ 本地网关 localhost:20128/v1；237 provider、90+ 免费 tier     │
│             │ 17 种路由策略 + auto-fallback + **内置 RTK→Caveman 压缩管**  │
│             │ 宣称堆叠后 15–95% token 省 + ~1.6B 免费 token/月（需实测）    │
└─────────────┴──────────────────────────────────────────────────────────────┘

管道顺序（OmniRoute README）：
  工具输出 → RTK 过滤/分组/截断 → 模型回复 → Caveman 精简规则 → 用户

### 替换你 S 仓里的什么

  SUNSET：codex_native_provider_scheduler_phase4（3408 行）
          provider_price_catalog · token budget gate · 手搓 width 晋级

  薄绑二选一（生产稳 vs 省钱实验）：
    A. LiteLLM — 成熟、文档全；routing/fallback/budget；XINAO 原地图默认
    B. OmniRoute — 免费 tier 聚合 + RTK/Caveman 已焊；营销数字要本地验证

  可叠加（不必二选一）：
    RTK 独立装 → 省工具输出 token（与网关无关）
    Caveman 独立装 → 省 Codex/Claude 啰嗦输出

### 是不是垃圾？

  RTK：真工具，单 Rust 二进制，有 benchmark — **保留，值得试**
  Caveman：真 skill；作者诚实说只省 output、skill 自身也吃 input — **有用，别当主链**
  OmniRoute：功能面宽（路由+压缩+MCP）；README 营销重、免费额度需 dashboard 实测
           — **不是垃圾，但只能当 L9/L8 薄网关接缝，不能替代 Temporal/LangGraph**

  警惕：免费 tier / 1.6B token 宣传 ≠ 你本地 invoke 已闭合；先小规模探活再 sunset 手搓调度。

### XINAO 薄绑接缝（≤5 件事）

  1. 所有 worker/agent 模型调用 → 单一 base_url（LiteLLM 或 OmniRoute /v1）
  2. Codex S shell 工具链 → 可选 rtk hook
  3. 长 readback/汇报 → 可选 Caveman 或 RTK json/summary
  4. rate limit / fallback / combo → 网关配置，不在 Python 里重写
  5. 证据 → D:\XINAO_RESEARCH_RUNTIME\provider\<run_id>.json（花了多少、走了哪路）
