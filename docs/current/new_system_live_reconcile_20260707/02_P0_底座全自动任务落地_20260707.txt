## 1. 状态分类

状态定义：

- `已实现`: 有默认入口、运行态证据、验证脚本或 same-wave 证据。
- `半实现`: 有代码/状态/测试，但未进入默认主线或缺 same-wave 证明。
- `已安装未绑定`: 依赖在 `.venv`，但没有运行态 latest/canary/default route。
- `手搓壳待替换`: 本地代码存在，但应由成熟组件承载核心能力。
- `未实现`: 只有文本目标，没看到代码/运行态证据。

## 3. 手搓替换原则

不是所有本地代码都要删。

应保留为领域 thin adapter：

- SourceLedger
- WorkerBrief
- ArtifactAcceptanceQueue
- AllocationPlan envelope
- Chinese readback generator
- S-specific boundary/verifier

应由 mature carrier 替换核心能力：

- durable orchestration -> Temporal。
- inner graph/checkpoint/interrupt -> LangGraph。
- model gateway/retry/cooldown/load balancing -> LiteLLM。
- experiment tracking -> MLflow。
- data version/lineage -> DVC/LakeFS/OpenLineage。
- data contract -> Pandera/Great Expectations。
- trace -> OpenTelemetry/Langfuse/Phoenix。
- API contract -> Schemathesis/Pact。
- optimization/search -> Optuna/Ray/Ax/ASHA/HyperBand。
- tool/capability discovery -> MCP/OpenAPI/Backstage/Docker MCP Gateway。

## 5. 工程落地模板

每个 mature carrier 替换都必须按同一模板落地，不能只写“建议替换”。

模板：

1. `replace_target`
   - 当前手搓壳或半实现对象。
2. `mature_carrier`
   - 采用的成熟组件和版本。
3. `thin_adapter`
   - 本仓只保留的最薄 adapter/schema/policy。
4. `default_mainline_binding`
   - 接入 RootIntentLoop / Temporal workflow / activity / ProviderScheduler / WorkerBrief / FanIn/AAQ 的具体点。
5. `runtime_evidence`
   - D runtime 写哪些 latest/record/jsonl。
6. `verification`
   - 哪个脚本或测试证明默认路会调用它。
7. `acceptance`
   - AAQ 接受什么，不接受什么。
8. `readback`
   - 中文 readback 必须写已接入、未接入、下一机器动作。
9. `fallback_or_blocker`
   - 不满足时写 named blocker，不写完成。

## 8. 工程态验收门禁

每个 P0 落地项必须同时满足：

1. `repo_binding`
   - 有代码、配置、脚本或 workflow 入口变更。
2. `default_mainline_binding`
   - 能说明进入哪个默认焊接点。
3. `runtime_invocation`
   - 真实调用过，不是只写文件。
4. `evidence`
   - D runtime 有 record/latest/jsonl。
5. `verification`
   - 有 pytest 或 verifier。
6. `fan_in_aaq`
   - 结果进入 FanIn/AAQ 或明确说明为何只是 P1 candidate。
7. `readback`
   - 中文 readback 写清已做/未做/下一机器动作。
8. `git`
   - repo 变更提交并推送。

不满足时，只能写 named blocker：

- `NO_ACTIVE_333_MAINLINE`
- `MATURE_CARRIER_INSTALLED_NOT_BOUND`
- `DEFAULT_MAINLINE_BINDING_MISSING`
- `FANIN_AAQ_MISSING`
- `RUNTIME_INVOCATION_MISSING`
- `VERIFIER_MISSING`
