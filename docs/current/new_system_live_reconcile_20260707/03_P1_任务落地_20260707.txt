## 3. P1 domain kernel

P1 需要的 domain 对象：

- ResearchEpisode。
- DatasetReceipt。
- DataContractFreeze。
- ValidityGateway。
- Hypothesis。
- ExperimentSpec。
- BaselineSpec。
- ExperimentRunnerPort。
- TrialLedger。
- StatisticalEvidence。
- PromotionGate。
- StrategyFamily。
- FailureTombstone。
- ReproBundle。
- NextFrontier。

这些对象属于 domain schema。它们不 import Temporal、LangGraph、MLflow、DVC、LiteLLM、Langfuse。成熟组件通过 ports/adapters 接入。

## 4. 正期望验证约束

必须有：

- lockbox。
- walk-forward / out-of-sample。
- null tournament。
- family-level budget。
- multiple-testing discount。
- trial-count penalty。
- failure tombstone。
- replay。
- repro bundle。
- promotion decision。

缺任何一个，都只能叫 toy/research candidate，不能叫正期望。

## 5. Grok 段审位置

Grok 段审保留，但只能作为 P1 SegmentReviewPort。

输入：

- source corpus。
- long text segment。
- experiment report。
- contradiction target。

输出：

- ClaimCard。
- contradiction。
- missing edge。
- replay case。
- audit note。

禁止输出：

- PASS。
- completion。
- stop decision。
- fact promotion。
- default owner。

## 6. P1 启动条件

P1 真实执行启动前，P0 必须满足：

1. active 333 mainline exists。
2. source package -> SourceLedger works。
3. WorkerBrief generated from source frontier。
4. ProviderScheduler/worker pool executes at least one same-wave lane。
5. FanIn/AAQ produces decision。
6. result_wait/readback no longer finalizes early。
7. mature carrier binding ledger exists。

否则 P1 只做 candidate/smoke/schema，不进入真实研究。
