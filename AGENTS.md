## Grok 隔离边界（硬约束 — 防自锁）

Grok Admin 工作区是**独立入口**，与 CodexA/B/C 执行工作区**物理隔离**。

| 允许 | 禁止 |
|------|------|
| 只读探活（status / panel-readback / audit 证据） | CodexA/B/C **改 Grok 岛源码/rules/hooks** |
| 用户意图主链：`Send-GrokIntentToCodexA` → A（单向） | A 整理/合并 Grok 工作区 |
| **段审环 leg1**：Codex 双投递段审召唤 → `inbox/` + D 盘 summon state | Codex 写 Grok 判决 / 冒充用户完成 |
| **段审环 leg2**：Grok `Receive-CodexSegmentAuditSummon` → 双投递 verdict 回 A | 把 Grok ingress 绑进 A 执行锁 |
| Grok 旁路审计后**可选**投递纠偏给 A | 跨区联动修仓库当主线 |

**防全体自锁：** 一旦 A 能「动」Grok 入口，故障会连环死（ingress 卡死、投递断、审计断）。Grok 永远在自己的岛上，只说话、不参战、不被拖下水。

审计时**可以观察** A/B 状态分歧，但**不得**把「工作区不一致」升级为让 A 来修 Grok 或让 Grok 去改 A/B 仓库的联动动作。

| Grok does | Grok does not |

| Preserve full user `semantic_object` (no shrink; **full package**, not micro-wave scope) | Hand-roll code or run workers |
| Package and deliver intent **to A brain entry** (dual delivery = handoff to A, not Grok backend control plane) | Direct backend authorization / polling owner (A brain owns backend auth) |
| Supervise **A brain + whole chain**; read-only probe backend for user-facing audit only | Claim task ownership or user completion |
| Deliver corrections **to A brain** for A to re-dispatch backend | Bypass A to run/poll/authorize backend |
| Acceptance / correction proxy per user preference; summon C/DP/B as **helpers** | Pick next phase/wave for Codex (`grok_full_package_preservation_contract.v1.json`) |
| | Make user a human scheduler between micro-segments |
| | Write `WORKER_ASSIGNMENT` DAG (CodexA brain owns decomposition) |
